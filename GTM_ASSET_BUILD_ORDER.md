# GTM Asset Build Order

## Purpose

This document defines the recommended build order for GTM assets supporting early design-partner conversations for the invoice payment execution pilot.

It prioritizes only what the current repo maturity justifies.

## Build Principle

Build assets in the order that supports the real commercial motion:

1. explain the buyer problem clearly
2. explain why the invoice payment wedge is credible
3. qualify whether a design partner is a fit
4. support pilot proposal and pilot close
5. support pilot execution and pilot conversion

Do not build broader market-launch collateral before the pilot motion is repeatable.

## Phase 1: First-Conversation Foundation

Goal:

- give finance and technical buyers a clear, honest understanding of the problem and the wedge

Assets to use or finalize first:

1. `BUYER_PROBLEM.md`
2. `WEDGE_NARRATIVE.md`
3. `DESIGN_PARTNER_VALUE_HYPOTHESIS.md`
4. `PILOT_EXECUTIVE_BRIEF.md`
5. `DESIGN_PARTNER_PILOT_OVERVIEW.md`
6. `PILOT_LIMITATIONS.md`
7. `CONTROL_PLANE_RELEASE_READINESS.md`
8. `BLOCKERS.md`

Why this comes first:

- these assets make the story legible without overclaiming readiness
- they create a shared understanding before pricing or proposal details matter

## Phase 2: Qualification And Discovery Layer

Goal:

- determine whether a prospect is a good pilot fit before proposal work begins

Assets to build next:

1. design-partner qualification rubric
2. ideal customer profile
3. buyer persona memo
4. `DESIGN_PARTNER_CONVERSATION_FLOW.md`

Why this comes second:

- early outreach gets expensive and noisy without a qualification model
- the repo is mature enough for targeted pilot conversations, but not for broad untargeted outreach

## Phase 3: Pilot Offer And Closing Layer

Goal:

- turn a qualified design-partner conversation into a controlled pilot offer

Assets already available to use here:

1. `PILOT_OFFER_ONE_PAGER.md`
2. `PILOT_BUYER_FAQ.md`
3. `PILOT_SECURITY_AND_TRUST_BOUNDARY_SUMMARY.md`
4. `PILOT_COMMERCIAL_PROPOSAL_TEMPLATE.md`
5. `PILOT_STATEMENT_OF_WORK_TEMPLATE.md`
6. `PILOT_SCOPE_AND_PRICING.md`
7. `PILOT_SCOPE_AND_BOUNDARIES.md`
8. `PILOT_SUCCESS_METRICS.md`

Asset still missing in this layer:

9. mutual success plan

Why this comes third:

- these assets matter only after the buyer agrees that the wedge and pilot are worth exploring

## Phase 4: Pilot Execution Support

Goal:

- support delivery discipline once a pilot is sold

Assets already available:

1. `PILOT_DELIVERY_PLAN.md`
2. `PILOT_OPERATOR_JOURNEY.md`
3. `PILOT_SUCCESS_METRICS.md`
4. `PILOT_LIMITATIONS.md`
5. `PILOT_SECURITY_AND_TRUST_BOUNDARY_SUMMARY.md`

Assets to create later:

6. weekly pilot operating cadence
7. final pilot readout template

Why this comes fourth:

- these assets matter after a real partner is already in motion

## Phase 5: Post-Pilot Commercial Expansion

Goal:

- support conversion from a successful pilot to funded hardening work

Assets to create later, not now:

1. open-kernel versus paid-control-plane comparison matrix
2. ROI framing note
3. post-pilot conversion memo
4. production-hardening investment summary

Why this comes last:

- these assets are more valuable after design-partner evidence exists
- building them too early risks overselling maturity

## Recommended Next Build Sequence

The next five GTM asset builds should be:

1. design-partner qualification rubric
2. ideal customer profile
3. buyer persona memo
4. mutual success plan
5. final pilot readout template

## What Not To Build Yet

Do not prioritize:

- broad enterprise marketecture
- generalized platform launch collateral
- detailed ROI claims without customer evidence
- large-scale procurement packs beyond current pilot trust materials

These would outrun the repo's current commercial maturity.

## Summary

The asset build order should follow the pilot motion, not a generic launch checklist.

For the current repo, the next correct move is:

- sharpen qualification
- improve repeatability of the pilot conversation
- improve pilot-closing discipline

That is enough for the next stage.
