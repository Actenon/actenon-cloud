# Commercial README Addendum

## Purpose

This addendum helps a commercial or design-partner reader navigate the most relevant materials in this repository.

The main `README.md` remains repo-oriented. This file is the shortest path through the commercial and pilot materials.

## What This Repo Is

Actenon Cloud is the commercial control plane for governed invoice payment execution.

It is not:

- the open kernel
- the verifier
- a production-ready payment execution platform

The current commercially credible motion is a controlled design-partner pilot around one outbound invoice payment workflow.

## Commercial Model In Brief

The current working commercial model is:

- a fixed pilot or setup fee for workflow mapping, configuration, validation, and supervised delivery
- later live operational pricing, if the pilot proves value, tied to proved and allowed consequential actions
- no usage charge for blocked, denied, or prevented actions

This is a working pilot-stage model, not final public pricing policy.

## Boundary Reminder

This repo governs the workflow around invoice payment execution. It may issue bounded proofs and track proof-related state, but proof verification remains external through a separate verifier repo or verifier interface.

That boundary matters for technical truth, but the commercial evaluation should start with the buyer problem and the pilot wedge, not with verifier internals.

## Best Short Description

The shortest honest description is:

Actenon Cloud is a commercial governance and traceability layer for one narrow invoice payment workflow, offered first through a controlled design-partner pilot.
